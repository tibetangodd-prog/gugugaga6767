# 糖宣人機

一個部署在 GitHub Pages 的純前端聊天網站。判斷使用者意圖的部分用**大量關鍵字比對**來做，不呼叫任何 AI API，**不需要任何 API 金鑰、不需要網路請求**，打開網站立刻就能用，也不會有額度或金鑰失效的問題。

版面已針對手機螢幕優化（安全區域、避免輸入框被 iOS 自動放大、避免整頁橡皮筋彈動），在桌面瀏覽器打開也會置中顯示成手機寬度。

## 運作方式

1. 使用者輸入一句話。
2. 如果這句話包含「煞筆」或「傻逼」，直接回嗆「你才煞筆」／「你才傻逼」。
3. 否則依序檢查這句話有沒有命中下面四種關鍵字清單中的任何一個：
   - 命中「感謝」關鍵字 → 補客氣
   - 命中「要求做某事」關鍵字 → 窩補藥
   - 命中「誇獎」關鍵字 → 🦀🦀
   - 都沒命中 → 隨機回覆「好強」「？」「咕咕嘎嘎」「阿巴阿巴」其中一句

如果一句話同時命中多種關鍵字（例如「謝謝你幫我查」同時有感謝跟要求），判斷順序是**感謝 > 要求做某事 > 誇獎**，所以這句會被判斷成感謝。

打開網站的第一句開場白固定是「泥豪」。

## 本機預覽

直接用瀏覽器打開 `index.html` 即可使用，不需要安裝套件、不需要啟動伺服器、不需要設定任何東西。

## 部署到 GitHub Pages

1. 在 GitHub 建立一個新的 repository（Public）。
2. 把 `index.html`、`style.css`、`script.js` 三個檔案推送到 repository 的根目錄。
3. 進入該 repository 的 **Settings → Pages**。
4. Source 選 **Deploy from a branch**；Branch 選 `main`，資料夾選 `/ (root)`，按 **Save**。
5. 等待約 1 分鐘，打開 GitHub 給的網址（格式類似 `https://你的帳號.github.io/repo名稱/`），就能看到「糖宣人機」上線，任何人打開都能直接聊天。

## 可以調整的地方（都在 `script.js` 最上方）

- `INSULT_PATTERNS`：互嗆關鍵字跟對應的回嗆文字，格式是 `{ keyword: "...", reply: "..." }`，要新增就照這個格式加一筆。
- `THANKS_KEYWORDS` / `REQUEST_KEYWORDS` / `PRAISE_KEYWORDS`：三個意圖各自的關鍵字陣列，都已經放了大量常見說法（包含口語、注音縮寫、簡單英文），但語言千變萬化，難免會漏，發現使用者常講某句話卻沒被正確判斷，直接把那句話拆成關鍵字加進對應的陣列即可，不需要改任何其他程式碼。
- `buildReply()`：四種意圖各自對應的固定回覆文字，other 情境的隨機選項也在這裡。

## 已知限制

純關鍵字比對沒有真正的語意理解，遇到反諷、雙重否定、或關鍵字清單沒覆蓋到的講法，可能會判斷錯誤或掉到「其他」分類，這是這個做法本身的取捨：換來的是零設定、零金鑰、零延遲、不會因為外部服務變動而壞掉。如果之後想換成真正理解語意的 AI 判斷，關鍵字清單可以直接保留下來當作分類失敗時的備援機制。
